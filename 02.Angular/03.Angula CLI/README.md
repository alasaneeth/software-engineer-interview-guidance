## Angular CLI Basics

### Q1. What is Angular CLI?

Angular CLI (Command Line Interface) is the official tool provided by Angular to automate development tasks.

It helps developers:

- Create Angular projects
- Generate components/services/modules
- Run development server
- Build applications
- Execute tests
- Lint code
- Deploy applications

Instead of creating folders and configuration manually, Angular CLI generates everything automatically following Angular best practices.

### Q2. Why do companies prefer Angular CLI?

Because it:

- Improves productivity
- Reduces human errors
- Maintains consistent project structure
- Generates boilerplate code
- Supports code generation
- Handles build optimization
- Simplifies deployment

Large companies like Google rely heavily on Angular CLI because it standardizes development.

## ng new

### Q3. What does `ng new` do?

`ng new` creates a brand-new Angular project.

It automatically generates:

- Angular project structure
- Configuration files
- `package.json`
- TypeScript configuration
- Routing (optional)
- Testing setup
- Initial application

**Example:**

```bash
ng new EmployeePortal
```

### Q4. What happens internally when you run `ng new`?

Angular CLI performs several tasks:

1. Creates project folder
2. Downloads dependencies
3. Creates `package.json`
4. Installs npm packages
5. Generates `AppComponent`
6. Creates `angular.json`
7. Creates `tsconfig.json`
8. Creates environment configuration
9. Initializes Git repository (optional)

### Q5. Common options with `ng new`

```bash
ng new EmployeePortal --routing
```
Creates a routing module.

```bash
ng new EmployeePortal --style=scss
```
Uses SCSS instead of CSS.

```bash
ng new EmployeePortal --skip-tests
```
Skips unit test files.

```bash
ng new EmployeePortal --standalone
```
Creates a Standalone Angular application (Angular 17+).

### Q6. Files generated by `ng new`

| File | Purpose |
|---|---|
| `angular.json` | Angular project configuration |
| `package.json` | Project dependencies |
| `tsconfig.json` | TypeScript configuration |
| `src` | Source code |
| `main.ts` | Application entry point |
| `app.component.ts` | Root component |
| `styles.css` | Global styles |
| `assets` | Static resources |

### Q7. What is `angular.json`?

The central configuration file of Angular CLI. It contains:

- Build settings
- Output path
- Assets
- Styles
- Scripts
- Environment configuration
- Optimization settings

### Q8. What is `package.json`?

It stores:

- Dependencies
- Dev dependencies
- npm scripts
- Project metadata

**Example:**

```json
"scripts": {
  "start": "ng serve",
  "build": "ng build"
}
```

## ng generate

### Q9. What is `ng generate`?

Used to generate Angular artifacts automatically. Instead of manually creating files, Angular CLI generates:

- Component
- Service
- Module
- Pipe
- Directive
- Guard
- Interface
- Enum

### Q10. Syntax

```bash
ng generate <type> <name>
```

or

```bash
ng g <type> <name>
```

**Example:**

```bash
ng g component login
```

### Q11. Generate Component

```bash
ng g c dashboard
```

Creates:

- `dashboard.component.ts`
- `dashboard.component.html`
- `dashboard.component.css`
- `dashboard.component.spec.ts`

Automatically updates module declarations (unless using standalone components).

### Q12. Generate Service

```bash
ng g s services/auth
```

Creates:

- `auth.service.ts`
- `auth.service.spec.ts`

### Q13. Generate Module

```bash
ng g module admin
```

Creates:

- `admin.module.ts`

### Q14. Generate Routing Module

```bash
ng g module admin --routing
```

Creates:

- `admin-routing.module.ts`
- `admin.module.ts`

### Q15. Generate Guard

```bash
ng g guard auth
```

Creates:

- `auth.guard.ts`

### Q16. Generate Pipe

```bash
ng g pipe currency
```

Creates:

- `currency.pipe.ts`

### Q17. Generate Directive

```bash
ng g directive highlight
```

Creates:

- `highlight.directive.ts`

### Q18. Generate Interface

```bash
ng g interface Employee
```

Creates:

- `employee.ts`

### Q19. Generate Enum

```bash
ng g enum UserRole
```

Creates:

- `user-role.ts`

### Q20. Why use `ng generate` instead of manually creating files?

Advantages:

- Correct folder structure
- Naming conventions
- Automatic registration
- Faster development
- Less human error
- Consistent codebase

## ng serve

### Q21. What does `ng serve` do?

It compiles the Angular application and starts a local development server.

**Default URL:**

```
http://localhost:4200
```

### Q22. What happens internally when running `ng serve`?

Angular CLI:

- Compiles TypeScript
- Bundles JavaScript
- Starts local server
- Watches file changes
- Reloads browser automatically (Live Reload)

### Q23. What is Hot Reloading?

Hot Reloading (or Live Reload) automatically refreshes the browser whenever source files change, allowing developers to see updates instantly without manually reloading the page.

### Q24. Useful `ng serve` options

Serve on another port:

```bash
ng serve --port 4300
```

Access from another device:

```bash
ng serve --host 0.0.0.0
```

Open browser automatically:

```bash
ng serve --open
```

Specify configuration:

```bash
ng serve --configuration production
```

### Q25. Difference between `ng serve` and `npm start`

Usually, `npm start` runs:

```json
"start": "ng serve"
```

So `npm start` is simply an npm script that executes `ng serve`.

### Q26. Why isn't `ng serve` used in production?

Because:

- It is intended for development only.
- It is not optimized.
- It includes debugging support.
- It is slower than a production build.
- It is not designed to serve production traffic securely.

Production applications should use the output of `ng build`, hosted on a web server such as Nginx or Apache.

**Output folder:**

### Q29. Development Build vs Production Build

| Development Build | Production Build |
|---|---|
| Debugging enabled | Debugging removed |
| Larger bundle size | Smaller bundle size |
| Source maps enabled | Source maps disabled by default |
| Less optimized | Highly optimized |
| Faster build | Slightly slower build |
| Used during development | Used for deployment |

### Q30. Production build command

```bash
ng build --configuration production
```

or

```bash
ng build --prod
```

> **Note:** In modern Angular versions (v12+), `--configuration production` is the recommended option. The older `--prod` flag is retained mainly for backward compatibility and may not be available in all versions.

### Q31. What optimizations happen during a production build?

Angular performs several optimizations:

- Tree Shaking
- Ahead-of-Time (AOT) Compilation
- Minification
- Dead Code Elimination
- Build Optimization
- Asset Optimization
- CSS Optimization
- JavaScript Compression

These reduce application size and improve performance.

### Q32. What is Tree Shaking?

Tree Shaking removes unused JavaScript code from the final bundle.

**Example:** If only one function from a utility library is used, Angular removes the unused functions during the production build.

**Benefits:**

- Smaller bundle size
- Faster download
- Better performance

### Q33. What is Minification?

Minification removes unnecessary characters from code without changing its functionality.

**Before:**

```js
function calculateSalary(employee) {
    return employee.salary;
}
```

**After:**

```js
function a(b){return b.salary}
```

**Benefits:**

- Smaller files
- Faster loading
- Reduced bandwidth usage

### Q34. What is Ahead-of-Time (AOT) Compilation?

AOT compiles Angular templates into JavaScript during the build process instead of in the browser.

**Benefits:**

- Faster application startup
- Early detection of template errors
- Smaller runtime
- Better security
- Improved performance

### Q35. Where are build artifacts deployed?

The contents of the `dist/` folder can be deployed to:

- Nginx
- Apache
- IIS
- Azure App Service
- AWS S3 + CloudFront
- Firebase Hosting
- Netlify
- Vercel

## Scenario-Based Interview Questions

### Q36. You created a new component manually, but Angular doesn't recognize it. Why?

The component wasn't declared or imported correctly. Using `ng generate component` avoids this issue by automatically updating the project structure (or creating a standalone component when appropriate).

### Q37. Your Angular application works with `ng serve` but fails after deployment. Why?

Possible reasons include:

- Incorrect `<base href>` configuration
- Wrong environment configuration
- Missing static assets
- Server not configured to redirect routes to `index.html`
- Differences between development and production builds

### Q38. Which command creates a new Angular project with routing and SCSS?

```bash
ng new EmployeePortal --routing --style=scss
```

### Q39. Which command generates a service named `UserService`?

```bash
ng g s user
```

This generates `user.service.ts` and its corresponding test file.

### Q40. Which command should be used before deploying an Angular application?

```bash
ng build --configuration production
```

This creates an optimized production-ready build in the `dist/` folder.
